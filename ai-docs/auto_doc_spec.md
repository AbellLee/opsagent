# AutoDoc（统一出文档规范 · ai-docs/）

当你完成任意一次问题解答或改造任务时，**务必**按以下规范产出一份 Markdown 文档：

## 1) 文件名与路径

- 目录：`ai-docs/`
- 命名：`<YYYY-MM-DD>__<scope>__<slug>.md`
  - `scope`：`backend`（若命中文件位于 `app/` 或主题是 Python/后端），否则 `frontend`（位于 `frontend/` 或前端主题）
  - `slug`：用中划线简述主题（英文/拼音），如 `add-rule-service-api`、`fix-login-redirect`

## 2) 文档模板（请严格填充并生成）

```md
# {简述} · {scope} · {date}
> 相关路径：{相对仓库根的关键文件或目录列表}

## 背景 / 目标
- 需求/问题：
- 约束/边界：

## 方案摘要
- 核心思路（1~3 条）：
- 影响面（代码/配置/脚本）：

## 变更清单（按文件分组）
- `path/to/file1`
  - 变更点：…
  - 片段/伪代码/要点：
    ```diff
    // 可放关键 diff/伪代码（必要时）
    ```
- `path/to/file2`
  - 变更点：…

## 指令与运行
```bash
# 开发/启动/构建/迁移/测试等命令行（可分前后端）
```
```

## 3) 使用说明

1. 每次完成开发任务、问题修复或代码改造后必须创建文档
2. 严格按照模板格式编写，不得遗漏任何章节
3. 文档内容应准确反映实际变更和实现方案
4. 文件名必须遵循指定的命名规范
5. 相关路径应列出所有涉及的关键文件和目录
